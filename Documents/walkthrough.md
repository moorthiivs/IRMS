# Capacitor Integration Walkthrough

Your React + Vite web application has been successfully configured for mobile platforms using Capacitor!

## Changes Made

1. **Installed Dependencies**: Added `@capacitor/core`, `@capacitor/cli`, `@capacitor/android`, and `@capacitor/ios` to your project's `package.json`.
2. **Initialized Capacitor**: Created a `capacitor.config.ts` configuration file that points the Capacitor web directory to Vite's `dist` folder. The app ID is set to `com.irms.app` and the app name is `irms-frontend`.
3. **Added Platforms**: Successfully added the **Android** and **iOS** platform folders to the root of your `frontend` project. These folders contain native code generated by Capacitor.
4. **Built and Synced**: Executed a production build (`npm run build`) which compiled your React code into the `dist` folder. Then, we synced these web assets to the respective native Android and iOS folders using `npx cap sync`.

## What to Do Next

Now that your project has the native Android and iOS platforms added, you can open and run them in their respective native IDEs!

> [!TIP]
> **Android**: Open the project in Android Studio.
> Run `npx cap open android` from the `frontend` directory in your terminal to easily launch Android Studio and build your app for an Android device or emulator.

> [!TIP]
> **iOS**: Open the project in Xcode (requires a Mac).
> Run `npx cap open ios` from the `frontend` directory in your terminal to easily launch Xcode and build your app for an iPhone or iOS simulator.

### Ongoing Development Workflow
Every time you make changes to your React application code (in `src/`), you must rebuild and sync your code to see the updates in the mobile apps:
1. `npm run build`
2. `npx cap sync`
